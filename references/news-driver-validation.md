# News Driver Validation

Use this reference when explaining revenue change, major projects, operating recovery, or strategic shifts with the help of news or public reports.

## Goal

Use news and public reporting to enrich context without letting recent articles, vague summaries, or wrong dates distort financial interpretation.

## Core rule

News can help explain *what may be driving change*. Official filings should still anchor *what the reported financial result is*.

## Required checks

### 1. Event date vs publication date
- [ ] Record the event date when possible.
- [ ] Record the publication / reporting date.
- [ ] Do not use a later article to explain an earlier period unless the timing clearly fits.

### 2. Financial-period relevance
- [ ] Ask whether the event plausibly affected the reported period.
- [ ] Distinguish among:
  - event already impacting current results
  - event announced but not yet financially visible
  - event relevant only to future periods

### 3. Source type
- [ ] Prefer official company releases, HKEX disclosures, investor presentations, or mainstream reporting with identifiable facts.
- [ ] Do not rely on anonymous aggregation or purely opinionated commentary.

### 4. Driver mapping
- [ ] If a news item is used to explain revenue or segment change, say exactly which segment it relates to.
- [ ] Avoid broad statements like "this project drove revenue" unless the timing and segment linkage are supportable.

### 5. Delivery wording
- [ ] When support is strong, write it as an evidence-backed driver.
- [ ] When support is partial, downgrade wording to:
  - "may have supported"
  - "is consistent with"
  - "provides context for"

## Good use cases
- major line openings or extensions
- cross-border traffic recovery
- airport / tourism recovery background
- project pipeline updates relevant to medium-term capex or financing context

## Bad use cases
- using recent media commentary as if it directly proves current-period revenue
- mixing future project narrative into current-period earnings conclusions without a timing note
