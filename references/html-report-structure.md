# HTML Report Structure

Use this reference when the user asks for a deliverable HTML report rather than rough notes.

## Recommended section order

1. Title block
2. Executive summary
3. Source and period scope
4. Business / operating-model overview
5. Revenue and segment analysis
6. Profitability and driver analysis
7. Balance sheet, leverage, and financing interpretation
8. Peer comparison when relevant
9. Key takeaways and risks
10. References

## Minimum requirements

- State the company name, ticker, currency, and covered period near the top.
- State which official documents were used.
- State whether the report uses FY, H1, quarter, TTM, or point-in-time data.
- Keep references visible and specific enough that a reviewer can trace each major claim.
- Keep charts self-contained and readable without external assets.

## Executive summary pattern

Keep the summary short and evidence-led:
- What the company is
- What drives the business model
- What changed in the requested period
- What the main conclusion is
- What caveat or limitation matters most

## Evidence block pattern

For judgment-heavy sections, prefer a compact pattern:
- Claim
- Supporting figures
- Source document / period
- Why it matters

## Reference style

Prefer a visible list containing:
- document name
- reporting period
- issuer / IR source
- link if available

## Chart rules for delivered HTML

- Use one chart per question.
- Put unit and period in the title or subtitle.
- Add a warning note when metrics are shown together for context but are not directly comparable.
- If chart readability is weak, replace it with a compact table.
