# v2 → v3 Acceptance Checklist

Use this checklist when validating whether a revised report really addresses the reviewer feedback captured in the MTR v2 notes. Do not stop at a binary yes/no check. For each section, judge whether the report is **not done**, **partially done**, **done**, or **done well**.

## 1. Language and final-draft polish

### Not done
- Narrative, headings, chart labels, and page packaging still feel like an analysis memo or half-translated draft.
- English abbreviations and analyst-style labels remain exposed throughout the deliverable.

### Partially done
- Main narrative has been localized into Chinese, but page titles, top-page labels, KPI labels, chart titles, or lead paragraphs still read like internal analysis packaging.
- Too many English abbreviations remain exposed without enough Chinese lead terms.

### Done
- Main narrative, page titles, headings, KPI labels, and chart titles are consistently localized.
- The report reads like a finished Chinese deliverable rather than a draft analysis memo.

### Done well
- Terminology is localized without becoming awkward.
- Page titles carry judgment, not just packaging language.
- Lead paragraphs are concise, client-facing, and clearly written.

## 2. Driver depth

### Not done
- Explanations stop at "costs increased", "margin declined", or other surface descriptions.

### Partially done
- Some drivers are named, but important changes still rely on generic explanations.

### Done
- Major revenue, margin, leverage, or profit changes are tied to specific disclosed drivers where available.

### Done well
- The report explains not only the named driver, but also why it matters to the overall conclusion and what it implies about earnings quality or business structure.

## 3. Comparability discipline

### Not done
- Different metric levels are compared as if they were directly comparable.

### Partially done
- Mixed metrics remain in the same section or figure, but some warnings are added.

### Done
- Like-for-like comparability is enforced in charts, tables, and judgments.
- Mixed metrics appear only when clearly labeled as explanatory context.

### Done well
- Each exhibit answers one question cleanly, with clear metric-level discipline and minimal reviewer attack surface.

## 4. Financial dashboard completeness

### Not done
- The so-called overview page contains only a small subset of metrics while implying completeness.

### Partially done
- A useful set of metrics has been added, but the panel still lacks important P&L, cash-flow, or balance-sheet items and does not clearly disclose the gap.

### Done
- The panel broadly meets the minimum dashboard standard or clearly states what is still missing.

### Done well
- The dashboard supports the report’s profit-quality, cash-generation, leverage, and return conclusions rather than acting as a decorative metric list.

## 5. Source breadth and period discipline

### Not done
- The report relies too narrowly on one filing or mixes disclosure timing with covered period.

### Partially done
- More sources are used, but period mapping is still vulnerable or insufficiently explicit.

### Done
- Annual report, interim results, announcements, and other official materials are used appropriately.
- Disclosure date is kept separate from covered period.

### Done well
- The report uses the latest reliable official period for current analysis while preserving a clean baseline and explicit period map.

## 6. Operating-model explanation

### Not done
- The report mostly lists segments or numbers without explaining business logic.

### Partially done
- Some business-model explanation is present, but the linkage among traffic, monetisation, recurring income, cyclical profit, and financing structure remains incomplete.

### Done
- The operating model is explained as a linked system rather than a list of segments.

### Done well
- The reader can see clearly how the company creates value, where recurring earnings come from, what amplifies reported profit, and how financing structure changes interpretation.

## 7. Peer comparison readiness

### Not done
- No peer framework is present where peer context is clearly needed.

### Partially done
- Peers are mentioned, but comparison purpose, metric alignment, or business-model insight is weak.

### Done
- The report defines the peer-comparison purpose and uses like-for-like framing where feasible.

### Done well
- Peer comparison materially improves the reader’s understanding of the target company’s distinctiveness, constraints, or strengths.

## 8. Reader clarity

### Not done
- The report assumes too much prior knowledge.

### Partially done
- Some terms are explained, but the logic still feels specialist-heavy.

### Done
- Key metrics and claims are understandable to a non-specialist reader.

### Done well
- The reader can follow the full chain from business model to metric selection to conclusion without specialist background.

## 9. Final acceptance

A revised report should normally be judged as follows:

- **Still below standard**: several sections remain at not done / partially done.
- **Deliverable but not final-final**: most core sections are done, but dashboard completeness or final-draft polish still lags.
- **Strong final draft**: core sections are done, with at least some sections done well, and no major reviewer-sensitive weakness remains.
- **Excellent final draft**: the report is defensible on language, metrics, period discipline, operating model, and evidence depth, with very limited reviewer attack surface.

Do not call the report a true final draft merely because it is much better than v2. Judge it against the actual checklist standard.
