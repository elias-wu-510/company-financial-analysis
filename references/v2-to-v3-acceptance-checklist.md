# v2 → v3 Acceptance Checklist

Use this checklist when validating whether a revised report really addresses the reviewer feedback captured in the MTR v2 notes.

## 1. Driver depth

- [ ] Do not stop at "costs increased" or "margin declined".
- [ ] Name the specific cost or balance-sheet driver where disclosed.
- [ ] If staff cost is the driver, add scale, ratio, or context where possible.
- [ ] Explain why the driver matters to the conclusion.

## 2. Comparability discipline

- [ ] Do not compare different metric levels as if they were directly comparable.
- [ ] If mixed metrics appear for context, add a visible warning.
- [ ] Keep one chart or table focused on one clear analytical question.

## 3. Unit, label, and language consistency

- [ ] Use one main narrative language throughout.
- [ ] Use one unit convention throughout.
- [ ] Use one period-labeling convention throughout.
- [ ] Check chart titles, subtitles, legends, and tables for consistency.

## 4. Source breadth

- [ ] Do not rely only on the annual report when interim reports, results announcements, or relevant releases are needed.
- [ ] Distinguish official financial results from commentary or outlook.
- [ ] Keep disclosure date separate from covered period.

## 5. Peer comparison readiness

- [ ] If peer comparison is added, define the comparison purpose first.
- [ ] Keep peer metrics on a like-for-like basis.
- [ ] Explain what the peer comparison reveals about the target company’s operating model.

## 6. Reader clarity

- [ ] Define key metrics in plain language.
- [ ] Explain not only what the metric is, but why it matters.
- [ ] Check whether a non-specialist reader can follow the logic.

## 7. Final acceptance

- [ ] The revised report is meaningfully better than v2 on substance, not only wording.
- [ ] The report can be delivered together with a checked checklist or an honestly qualified checklist.
