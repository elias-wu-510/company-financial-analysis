# Report Style — Light Theme

Use this reference when producing deliverable HTML reports for company analysis.

## Goal

Default to a light, reading-friendly report style rather than a heavy dark presentation style.

## Style direction

Prefer:
- light background
- dark grey text instead of harsh pure-black blocks
- soft section backgrounds
- restrained accent colours
- charts that remain readable when printed or exported to PDF

Avoid:
- large dark panels as the dominant visual style
- high-contrast neon accents
- overly saturated chart palettes
- designs that feel like dashboards rather than readable reports

## Visual rules

### Page background
- use white or very light neutral backgrounds
- keep strong colour blocks limited to small highlights

### Text
- prefer dark grey body text
- keep heading contrast clear but not visually aggressive
- maintain long-form reading comfort

### Cards and callouts
- use pale grey / pale blue / pale beige backgrounds
- reserve stronger colour for warning notes or emphasis only

### Charts
- prefer muted blue / teal / slate / sand palettes
- keep colours distinguishable but not loud
- ensure exported PDFs still look clean in grayscale-like reading conditions

## Delivery rule

Unless the user explicitly asks for a dark theme, default all report templates and newly generated HTML deliverables to a light theme.
