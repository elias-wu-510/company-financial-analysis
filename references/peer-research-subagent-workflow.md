# Peer Research via Subagent Workflow

Use this reference when a company-analysis task needs peer comparison or industry positioning and the peer-research workload would otherwise overload the main report-writing flow.

## Goal

Split the work into:
1. **subagent peer research**
2. **main-agent report writing**

This keeps the main report focused and reduces failure risk from long mixed workflows.

## Recommended division of labor

### Subagent
Responsible for:
- identifying suitable peers
- locating official IR / filings / presentations
- summarising business-model positioning
- noting what is comparable and what is not
- producing a short peer note

### Main agent
Responsible for:
- the target company’s main report
- source / period discipline
- integrating the peer note into the report
- charts, checklist, and client-delivery cleanup

## Output contract for the subagent

Prefer a compact peer note containing:
- selected peers
- why they were chosen
- official sources used
- comparable dimensions
- non-comparable dimensions / warnings
- short conclusion on the target company’s position, strengths, weaknesses, and sustainability questions

## When to use this workflow

Use it by default when:
- the user asks for peer comparison
- the user asks about industry position or relative strengths
- peer research needs multiple companies or multiple IR sites
- the main report is already long and should stay stable

## Delivery rule

The subagent should not try to write the entire final report. It should provide the peer-research layer so the main agent can integrate it cleanly.
